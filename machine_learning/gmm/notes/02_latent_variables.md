# 02 — Latent Variables

---

## 1. What is a Latent Variable?

A **latent variable** is a variable that participates in the probabilistic model but is **never observed** in the data.

In a GMM:
- $\mathbf{x}$ — observed data point
- $z \in \{1, \ldots, K\}$ — latent component assignment ("which Gaussian generated this point?")

We observe $\mathbf{x}$, but we never know $z$. This is what makes GMM fitting hard.

---

## 2. 1-of-K Encoding

It is convenient to represent $z$ as a binary vector $\mathbf{z} \in \{0,1\}^K$ with exactly one entry equal to 1:

$$
z_k \in \{0, 1\}, \qquad \sum_{k=1}^{K} z_k = 1
$$

So "$z_k = 1$" means "this point was generated by component $k$".

---

## 3. Joint Distribution

The full joint distribution over observed and latent variables:

$$
p(\mathbf{x}, \mathbf{z} \mid \boldsymbol{\theta}) = p(\mathbf{z} \mid \boldsymbol{\pi}) \, p(\mathbf{x} \mid \mathbf{z}, \boldsymbol{\mu}, \boldsymbol{\Sigma})
$$

Expanding using the 1-of-K encoding:

$$
p(\mathbf{z} \mid \boldsymbol{\pi}) = \prod_{k=1}^{K} \pi_k^{z_k}
$$

$$
p(\mathbf{x} \mid \mathbf{z}, \boldsymbol{\mu}, \boldsymbol{\Sigma}) = \prod_{k=1}^{K} \mathcal{N}(\mathbf{x} \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)^{z_k}
$$

Therefore:

$$
\boxed{p(\mathbf{x}, \mathbf{z} \mid \boldsymbol{\theta}) = \prod_{k=1}^{K} \left[ \pi_k \, \mathcal{N}(\mathbf{x} \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k) \right]^{z_k}}
$$

---

## 4. Marginalization

The **observed-data likelihood** is obtained by summing the joint over all possible values of $\mathbf{z}$:

$$
p(\mathbf{x} \mid \boldsymbol{\theta}) = \sum_{\mathbf{z}} p(\mathbf{x}, \mathbf{z} \mid \boldsymbol{\theta}) = \sum_{k=1}^{K} \pi_k \, \mathcal{N}(\mathbf{x} \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)
$$

This recovers the GMM density from `01_gaussian_and_mixtures.md`.

---

## 5. Posterior over Latent Variables (Responsibility)

Given an observed $\mathbf{x}$, what is the probability it came from component $k$?

By Bayes' theorem:

$$
p(z_k = 1 \mid \mathbf{x}, \boldsymbol{\theta}) = \frac{p(z_k = 1) \, p(\mathbf{x} \mid z_k = 1)}{\sum_{j=1}^{K} p(z_j = 1) \, p(\mathbf{x} \mid z_j = 1)}
$$

$$
\boxed{r_{nk} \triangleq p(z_k = 1 \mid \mathbf{x}_n, \boldsymbol{\theta}) = \frac{\pi_k \, \mathcal{N}(\mathbf{x}_n \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)}{\sum_{j=1}^{K} \pi_j \, \mathcal{N}(\mathbf{x}_n \mid \boldsymbol{\mu}_j, \boldsymbol{\Sigma}_j)}}
$$

This quantity $r_{nk}$ is called the **responsibility** of component $k$ for data point $n$.

**Properties:**
- $r_{nk} \in [0, 1]$
- $\sum_{k=1}^{K} r_{nk} = 1$ for all $n$
- Computed in the **E-step** of EM (see `03_em_algorithm.md`)

---

## 6. Why Latent Variables Make Optimization Hard

If $z$ were observed, we could fit each Gaussian independently using closed-form MLE.

Because $z$ is **unobserved**, the log-likelihood contains a **sum inside a log**:

$$
\log p(\mathbf{X} \mid \boldsymbol{\theta}) = \sum_{n=1}^{N} \log \sum_{k=1}^{K} \pi_k \, \mathcal{N}(\mathbf{x}_n \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)
$$

The $\log \sum$ has no closed-form MLE solution. This is why we need EM. See `03_em_algorithm.md`.
